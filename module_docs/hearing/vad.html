hearing.vad

Voice-activity detection and utterance segmentation for the live pipeline.

VAD owns utterance boundaries (never fixed time-slicing, which cuts words and wrecks WER): detect speech -> buffer the utterance -> on trailing silence, finalize. segment_utterances() turns an async stream of audio blocks into an async stream of (utterance_samples, start_ms) pairs that the streaming STT engine transcribes one at a time.

Two detectors behind a tiny protocol:

  • EnergyVAD — dependency-free RMS-threshold VAD. The default; great for clean/loud meeting audio and fully deterministic (so tests need no models).

  • SileroVAD — neural VAD (accurate on noisy audio); optional, lazy.

See the hearing-live-pipeline skill for the rationale and the latency budget.

class hearing.vad.EnergyVAD(threshold: float = 0.01)[source]

RMS-energy threshold VAD — dependency-free default.

A block counts as speech when its RMS energy exceeds threshold. Simple and deterministic; for noisy real-world audio prefer SileroVAD. threshold is in the same units as the float audio (roughly [-1, 1]).

async hearing.vad.segment_utterances(frames: AsyncIterator[ndarray], *, sample_rate: int, vad: VAD | None = None, silence_ms: int = 700, min_speech_ms: int = 200, max_utterance_ms: int = 30000) → AsyncIterator[tuple[ndarray, int]][source]

Group an async stream of audio blocks into utterances.

Yields (utterance_samples, start_ms) once per finalized utterance: speech accumulates into a buffer; after silence_ms of trailing silence (or max_utterance_ms of continuous speech) the buffer is emitted with its start offset (integer ms from the beginning of the stream). Utterances with less than min_speech_ms of actual speech are dropped as noise.
