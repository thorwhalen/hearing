hearing.stt

Speech-to-text engines behind the hearing STT facade.

The rest of the architecture only ever sees the STTEngine Protocol: transcribe(audio) -> segments. Swap the engine (local Whisper for cloud Deepgram, say) by changing one line of assembly — nothing downstream knows or cares. See the hearing-stt skill for the full engine landscape, the per-minute pricing table, and the local-vs-cloud decision thresholds.

The default engine is FasterWhisperSTT (faster-whisper / CTranslate2): fast, CPU-friendly via int8, no PyTorch, runs well on Apple Silicon. Heavy deps are optional — transcribe raises an informative error telling you what to pip install if the engine isn’t available.

hearing.stt.DEFAULT_WHISPER_MODEL: str = 'base'

Default local model. “base” balances speed/quality; tests use “tiny”.

class hearing.stt.FasterWhisperSTT(model_size: str = 'base', device: str = 'cpu', compute_type: str = 'int8', word_timestamps: bool = False, vad_filter: bool = True, beam_size: int = 5, vad: 'VAD' | None = None)[source]

Local STT via faster-whisper (CTranslate2) — the default engine.

Parameters:

  • model_size – whisper size name (“tiny”, “base”, “small”, “medium”, “large-v3”, “large-v3-turbo”, …) or a CTranslate2 model path.

  • device – “cpu”, “cuda”, or “auto”.

  • compute_type – “int8” (default, lean) / “int8_float16” / “float16” / …

  • word_timestamps – emit per-Word timing (enables mid-utterance keyword triggers in the agent layer).

The model is loaded lazily on first use (functools.cached_property), so constructing the engine is cheap and import-time stays light.

async stream_transcribe(frames: AsyncIterator[ndarray], *, sample_rate: int) → AsyncIterator[TranscriptSegment][source]

Stream finalized segments as utterances complete (the live path).

VAD (self.vad, default EnergyVAD) groups the incoming frame blocks into utterances; each utterance is transcribed with the same batch transcribe and emitted as a FINALIZED segment (meta['final'] = True) with its span offset to absolute stream time.

Same Protocol method, same TranscriptSegment spine as batch — the live milestone adds no new interface (see hearing-live-pipeline).

transcribe(audio: ndarray, *, sample_rate: int, language: str | None = None) → Sequence[TranscriptSegment][source]

Transcribe a whole mono clip; return ordered segments.

audio is a 1-D float array. It is resampled to 16 kHz if needed, so callers may pass native-rate audio.

hearing.stt.default_engine(**kwargs) → FasterWhisperSTT[source]

Return the project’s default STT engine (faster-whisper).

Tests and quick runs typically pass model_size="tiny".
